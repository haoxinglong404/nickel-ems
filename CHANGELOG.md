## [v0.8] - 2026-04-19

### ✨ 新增
- **备件模块**：1777 条备件（1410 长周期 + 367 随机）
  - 四卡筛选：全部 / 长周期 / 随机 / 低库存（≤3）
  - 领用 / 入库两种操作
  - 完整的出入库历史记录
- **撤销/硬删除双轨制**（备件历史）：
  - 所有人：撤销自己的记录（生成反向记录）
  - 管理员：硬删除任何记录（同步调库存）
- **工单联系人字段**：
  - `contactName`（必填）+ `contactPhone`（必填）
  - 支持任意文本（手机号、企业微信、姓名备注）
  - 自动记忆上次填写，下次默认填充
- **"其他"设备选项**：
  - 工单选设备时可选"其他/非设备工作"
  - 用于管道、护栏、楼梯等非设备工作
  - 选中后 `equipmentId === 'other'`
- **翔总 VIP 欢迎卡**：
  - 工号 `6724013003` 登录时弹出"翔总辛苦了"欢迎卡片
  - 其他用户直接进入系统（无干扰）
- **用户管理支持"管理员"角色**：
  - 添加/编辑用户时可选"管理员"
  - 自我保护：不能改/删自己
  - 最后一管理员保护：不能降级/删除最后一个 admin
- **6 条历史工单自动导入**：
  - 从《高压酸浸车间需检修作业记录表.xlsx》导入
  - 全部状态 = 待审批，提报人 = 郝行龙
  - 联系人 = 马学斌，联系方式 = 企业微信
  - 1 条匹配设备（201-PE-PP-001D 原矿浆输送泵），5 条标"其他"

### 🔧 修复
- 欢迎卡去掉"敬"字，视觉更简洁
- 备件领用/入库表单的数字输入不再打断 reason 输入体验

### 📝 教训
- **v0.6 工单 switchModule bug** 的根因：`html.replace(old, new)` 在字符串不完全匹配时**静默失败**。解决方法：每次 `replace` 前 `assert old in html`。
- **字段名混乱**：v0.8 早期用了 `siteContact`/`siteContactInfo`，后来发现已有 `contactName`/`contactPhone`。**教训**：改代码前用 `grep` 看清楚现有字段名。

## [v0.6] - 2026-04-17

### ✨ 新增

  - 5 种状态：待审批 / 已通过 / 已拒绝 / 维修中 / 待确认 / 已完成 / 重开

  - 所有人可提报
  - 李主任/王主任单签审批
  - 师傅抢单 + 管理员指派
  - 提报人关单确认

### 🔧 修复
- `switchModule` 对工单 Tab 的支持
- `isMyTodo` 对提报人的判断逻辑

### 📝 教训
- **UI 跳转流程**：提报后应立即打开新工单详情，让用户看到"已成功提交"的视觉反馈。
- **乐观 UI**：不要等 Firestore 返回才切换页面，`cache.unshift` 先给用户看。

## [v0.5] - 2026-04-15

### ✨ 新增
- **Firebase Firestore 接入**：
  - 项目 `nickel-ems`，地区 `asia-southeast1`
  - 测试模式规则（~2026-05-19 到期）
  - SDK 12.12.0 ES Module 从 gstatic CDN 加载
- **设备台账云端化**：361 台设备自动上传
- **GitHub Pages 部署**：
  - 仓库 `haoxinglong404/nickel-ems`
  - URL: https://haoxinglong404.github.io/nickel-ems/
  - 授权域名已配置

### 🔧 重要决策
- **选择 Firestore 而非 Realtime DB**：更现代、查询更灵活
- **批量写入种子数据**：`writeBatch()` 每批最多 450（Firestore 限制 500）
- **实时订阅**：用 `onSnapshot` 而非轮询，所有客户端秒级同步

## 📐 版本号规则

- **主版本号**（1.x）：架构性变更（比如引入框架）—— 本项目长期不会有
- **次版本号**（0.X）：主要功能模块上线
- **修订号**（0.8.X）：bug 修复或小功能

**当前阶段**：v0 时代（MVP + 逐模块上线）。v1.0 里程碑 = Firestore 安全规则升级 + PWA 上线。
